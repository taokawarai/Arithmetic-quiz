# Feature Specification: WAIS-IV 算数サブテスト 自動問題生成エンジン

**Feature Branch**: `001-wais-arithmetic-generator`

**Created**: 2026-05-17

**Status**: Draft

**Input**: User description: "WAIS-IVの「算数（Arithmetic）」サブテストを模した問題を自動生成するプログラム"

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 問題の自動生成と即時取得 (Priority: P1)

開発者・アプリケーションは `generate_problem(level_id)` を呼び出すだけで、指定レベルに合った問題文・正解・単位を含むオブジェクトをその場で受け取ることができる。生成された問題は必ず「整数または合理的な小数」の解を持ち、「現実にあり得ない数値（負のお釣り、分数の個数など）」は一切含まれない。

**Why this priority**: 問題生成はシステムの中核機能であり、これが動かなければ他のすべての機能が成立しない。

**Independent Test**: `generate_problem("level1")` を 100 回連続で呼び出し、返却された全オブジェクトの `answer` が整数かつ問題文の文脈に合理的な値であることを検証できる。

**Acceptance Scenarios**:

1. **Given** レベル1を指定して `generate_problem("level1")` を呼び出す、**When** 関数が実行される、**Then** `{"question": "...", "answer": <整数>, "unit": "..."}` 形式のオブジェクトが返り、回答は必ず正の整数である
2. **Given** 制約条件ギリギリの変数値（A=B の加算など）が候補になる、**When** バリデーターが実行される、**Then** その値は棄却され再抽選が行われ、最終的に制約を満たす問題のみが出力される
3. **Given** レベル4の複合問題を生成する、**When** 関数が実行される、**Then** 逆算ロジックで生成された変数から問題文が組み立てられ、解が整数になっていることが保証される

---

### User Story 2 - 問題文の読み上げとカウントダウン (Priority: P1)

ユーザーが「開始」ボタンを押すと、問題文がブラウザの音声読み上げ機能（Web Speech API）で日本語読み上げされる。読み上げ終了後、そのレベルの制限時間（30秒または60秒）のカウントダウンが開始され、ユーザーは制限時間内に回答できる。

**Why this priority**: 問題文の提示方法はテストの中核であり、WAIS-IV を模倣するうえで音声読み上げは必須の機能である。

**Independent Test**: 問題生成後に開始ボタンを押し、問題文が日本語で読み上げられ、読み上げ終了後にカウントダウンが開始されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 問題が生成されテスト画面が表示されている、**When** ユーザーが「開始」ボタンを押す、**Then** 問題文が日本語でブラウザの音声読み上げ機能により読み上げられる
2. **Given** 問題文の読み上げが終了した、**When** 自動的に、**Then** そのレベルの制限時間（レベル1・2は30秒、レベル3・4は60秒）のカウントダウンタイマーが表示・開始される
3. **Given** カウントダウン中、**When** カウントダウンが 0 になるまでに回答が入力された、**Then** 回答を受け付けて採点画面に遷移する
4. **Given** カウントダウン中、**When** カウントダウンが 0 になっても未回答、**Then** 自動的に不正解として処理される
5. **Given** Web Speech API が利用できないブラウザでアクセスした、**When** 開始ボタンを押す、**Then** エラーメッセージが表示され、テストは実施しない

---

### User Story 3 - レベル別・サブカテゴリ別問題の取得 (Priority: P2)

利用者（テストアプリ・学習アプリなど）は難易度レベル（1〜4）を指定して問題を取得できる。レベルに対応する制限時間も出力に含まれることで、タイマー機能と連携できる。

**Why this priority**: WAIS-IV を模倣するうえで難易度段階は本質的な要件。ただし P1 が動いた後でレベル区分を加えても MVP は成立する。

**Independent Test**: `generate_problem("level2")` を呼び出し、返却された問題がレベル2の問題パターン（複数アイテム合計・平均・ダース換算など）のいずれかであり、`time_limit` が 30 秒であることを検証する。

**Acceptance Scenarios**:

1. **Given** レベル1〜4の各レベルIDを指定して問題を生成する、**When** 関数が実行される、**Then** 出力に `level` と `time_limit`（レベル1・2 は 30 秒、レベル3・4 は 60 秒）が含まれる
2. **Given** 存在しないレベルID（例: `"level9"`）を指定する、**When** 関数が実行される、**Then** エラーが明示的に返され、部分的に生成された問題オブジェクトは返らない

---

### User Story 4 - シチュエーション（皮）の多様化 (Priority: P3)

同じ計算ロジック（骨組み）に対して、複数のシチュエーション（買い物・工場・乗り物など）がランダムに割り当てられ、毎回異なる文脈の問題が生成される。アイテム名・登場人物名は辞書データから抽選される。

**Why this priority**: 問題の多様性は学習体験の質を高めるが、P1・P2 が動いた後でも段階的に追加できる。

**Independent Test**: 同じ計算骨組みで `generate_problem` を 10 回連続呼び出し、少なくとも 2 種類以上の異なるシチュエーション・アイテム名が出現することを確認する。

**Acceptance Scenarios**:

1. **Given** 乗算パターン（レベル1-2）の問題を複数回生成する、**When** 各問題文を確認する、**Then** 登場するアイテム名（りんご・みかんなど）が辞書から抽選され、問題ごとに異なる可能性がある
2. **Given** シチュエーション辞書に買い物・工場・乗り物の3種が登録されている、**When** 同一骨組みで問題を 3 回生成する、**Then** 少なくとも 2 種類のシチュエーションが使用される

---

### User Story 5 - サブカテゴリ指定による問題絞り込み (Priority: P4)

利用者は特定のサブカテゴリ（例: 速さ・割合・年齢算など）を指定して問題を生成できる。特定の苦手分野を集中的に練習したい場合に役立つ。

**Why this priority**: 利便性向上の追加機能。コアの生成エンジンが完成した後に追加実装可能。

**Independent Test**: `generate_problem("level3", sub_category="speed")` を呼び出し、速さ・時間・距離パターンの問題のみが返ることを確認する。

**Acceptance Scenarios**:

1. **Given** レベルとサブカテゴリを両方指定する、**When** 関数が実行される、**Then** 指定サブカテゴリに属する問題パターンのみから問題が生成される
2. **Given** 指定したサブカテゴリがそのレベルに存在しない、**When** 関数が実行される、**Then** 適切なエラーメッセージが返る

---

### Edge Cases

- 制約条件を満たす変数の組み合わせが存在しない場合（例: 除算で割り切れる A を探せない）、無限ループに入らず最大試行回数内でエラーを返す
- レベル4の逆算ロジックで逆算結果が整数にならない変数セットが選ばれた場合、再試行して有効な組み合わせを返す
- 問題文テンプレートに未定義の変数プレースホルダーが含まれる場合、テキストレンダリング前にエラーを検出する
- アイテム名・人物名辞書が空の場合、フォールバック値（例: "もの"、"Aさん"）を使用する
- 年齢算で計算結果が 0 以下になる（過去に遡る）ような変数の組み合わせは棄却する
- Web Speech API が利用できないブラウザでアクセスした場合、エラーメッセージを表示してテストを実施しない
- 読み上げ中にユーザーが回答操作を行った場合、読み上げ完了まで回答入力を無効にする
- 問題文に算用数字が含まれる場合でも、読み上げエンジンが正しく日本語（例：「3」→「さん」）で読み上げられることを保証する
- カウントダウンが 0 になったとき何も入力されていない場合、自動的に不正解として処理する

---

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは `generate_problem(level_id)` 関数を公開し、レベル ID（"level1"〜"level4"）を受け取って問題オブジェクトを返さなければならない
- **FR-002**: 問題オブジェクトは `question`（問題文文字列）、`answer`（数値）、`unit`（単位文字列）、`level`（レベル番号）、`time_limit`（制限時間秒数）を含まなければならない
- **FR-003**: システムは各問題パターンに対応するバリデーター（制約チェッカー）を持ち、制約を満たさない変数の組み合わせを自動的に棄却して再抽選しなければならない
- **FR-004**: 再抽選の最大試行回数は設定可能で、上限を超えた場合はエラーを返さなければならない（無限ループ禁止）
- **FR-005**: 解答（`answer`）は必ず整数、または現実的に意味のある合理的な小数（小数点以下1〜2桁まで）でなければならない
- **FR-006**: 負の値・ゼロが現実的に意味をなさない文脈（個数・枚数・人数・金額のお釣りなど）では、answer が 0 以下にならないことを保証しなければならない
- **FR-007**: システムはアイテム名（果物・文具・食べ物など）と登場人物名の辞書データを持ち、問題文テンプレートのプレースホルダーにランダムに割り当てなければならない
- **FR-008**: 同一の計算骨組みに対して複数のシチュエーション（買い物・工場・乗り物など）を持ち、問題生成時にランダムに選択しなければならない
- **FR-009**: サブカテゴリ指定（省略可能）を `generate_problem(level_id, sub_category=None)` の形でサポートしなければならない
- **FR-010**: レベル1・2 の制限時間は 30 秒、レベル3・4 の制限時間は 60 秒でなければならない
- **FR-011**: レベル4の逆算が必要な問題パターン（年齢算・入れ子割合・つるかめ算など）は、答えから逆算して変数を決定するロジックを持ち、常に整数解が保証されなければならない
- **FR-012**: 全問題パターンは `id`、`level`、`sub_category`、`time_limit`、`template`、`variables`、`constraints`、`formula` を持つ構造化オブジェクトとして定義されなければならない
- **FR-013**: 問題提示画面は「開始」ボタンを表示し、押下するとブラウザの音声読み上げ機能（Web Speech API）で問題文を日本語で読み上げなければならない
- **FR-014**: 読み上げ速度のデフォルトは **1.0**（スライダー初期値も 1.0）とし、ユーザーが変更できなければならない。読み方のデフォルトは **「区切る」**（句読点・助詞ごとに間を置く）とし、「つなげる」との切り替えが可能でなければならない
- **FR-015**: 問題文の読み上げ完了と同時に、その問題のレベルに対応する制限時間（30 秒または 60 秒）のカウントダウンタイマーを表示・開始しなければならない
- **FR-016**: カウントダウンタイマーは残り秒数を視覚的に表示し、0 秒になると自動的に不正解として処理しなければならない
- **FR-017**: 読み上げ中は回答入力操作を無効化し、読み上げ完了後に入力を有効化しなければならない
- **FR-018**: Web Speech API が利用できないブラウザでは、起動時にエラーメッセージを表示し、テストを開始できないようにしなければならない

### 問題パターンの制約仕様

#### レベル1 制約

| パターン | 変数範囲 | 制約条件 |
|---------|---------|---------|
| 1-1 加算 | A, B ∈ [2, 9] | A ≠ B |
| 1-1 減算 | A ∈ [6, 15], B ∈ [2, 9] | A > B |
| 1-2 乗算 | A ∈ [3, 9], B ∈ [3, 5] | なし |
| 1-2 除算 | B ∈ [2, 5], 商 ∈ [3, 9] | A = B × 商（割り切れる）|
| 1-3 お釣り | B ∈ {100, 500, 1000}, A は 10 の倍数 | B > A, B - A ≥ 20 |
| 1-4 時間 | A ∈ [1, 12], B, C は 5 の倍数 ∈ [5, 30] | B + C < 60 |
| 1-5 列の順番 | B, C ∈ [2, 9] | B ≠ C |

#### レベル2 制約

| パターン | 変数範囲 | 制約条件 |
|---------|---------|---------|
| 2-1 合計とお釣り | A, B は 10 の倍数, X, Y ∈ [2, 4], C ∈ {1000, 2000, 5000} | C > (A×X + B×Y) |
| 2-2 平均値 | A, B, C ∈ [60, 95] | (A+B+C) が 3 で割り切れる |
| 2-3 ダース | A ∈ [1, 3], B ∈ [2, 20] | A × 12 > B |
| 2-4 割合 | B ∈ {3, 4, 5}, A ∈ [12, 40] | A が B の倍数 |
| 2-5 過不足算 | B ∈ [3, 6], C ∈ [3, 8] | B×C - B < A < B×C |
| 2-6 年齢算 | X（年後）∈ [3, 10], C ∈ [2, 4], B ∈ [5, 15] | A = (B+X)×C - X, A ∈ [25, 50] |

#### レベル3 制約

| パターン | 変数範囲 | 制約条件 |
|---------|---------|---------|
| 3-1 割引 | A は 1000 の倍数, B ∈ [1, 4] | A × (1 - B/10) が整数（自動的に成立）|
| 3-2 速さ | B ∈ {15, 20, 30, 45} | A が (60/B) の倍数 |
| 3-3 旅人算 | X ∈ {60, 80}, Y ∈ {150, 200}, T ∈ {5, 10, 15} | (X×T) % (Y-X) = 0 |
| 3-4 比率と分配 | X, Y ∈ [1, 5] | gcd(X,Y)=1（互いに素）, A が (X+Y) の倍数 |
| 3-5 植木算 | A, B は正整数 | A が B の倍数 |
| 3-6 集合算 | A, B, C, D は正整数 | A - D < B + C（重複が存在）|

#### レベル4 制約

| パターン | 逆算ロジック | 制約条件 |
|---------|------------|---------|
| 4-1 速度複合 | 解を先に決め変数を逆算 | 解が正の整数 |
| 4-2 入れ子割合 | C → B → A の順で逆算 | C, (X+Y×(某余り)) がすべて整数になる切りのいい値 |
| 4-3 つるかめ算 | Y（100円枚数）, X（50円枚数）を決め A, B を算出 | (B - 50×A) が 50 で割り切れる |
| 4-4 濃度算 | A, B, C を決め解を算出 | (A×B) % 100 = 0, 解が整数または小数点1桁 |

### Key Entities

- **ProblemPattern**: 問題の骨組み。`id`, `level`, `sub_category`, `time_limit`, `template`, `variables`（変数定義と範囲）, `constraints`（制約関数リスト）, `formula`（解の計算式）を持つ
- **Situation**: 問題の皮。同一骨組みに差し替えられる文脈情報（背景文、アイテム種別、単位）のセット
- **VariableSet**: バリデーション済みの変数値の辞書。`ProblemPattern` に対してバリデーターが通過した組み合わせ
- **ProblemOutput**: 生成された問題の最終出力。`question`（文字列）, `answer`（数値）, `unit`（文字列）, `level`（整数）, `time_limit`（整数、秒）, `sub_category`（文字列）を持つ
- **ItemDictionary**: アイテム名辞書。カテゴリ（果物・文具・食べ物・動物など）ごとのアイテム名リスト
- **PersonDictionary**: 登場人物名辞書。名前のリスト（日本語名）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: `generate_problem` を 1000 回呼び出したとき、返却される全問題の answer が「整数または小数点以下1〜2桁以内の合理的な小数」であり、エラー率が 0.1% 未満である
- **SC-002**: 制約違反の変数が生成されたとき、バリデーターが 100% の確率でそれを検出して棄却する（偽陰性ゼロ）
- **SC-003**: `generate_problem` の実行時間が 1 回あたり平均 100ms 以内である（再抽選が最大試行回数の上限に当たるケースを除く）
- **SC-004**: 同一の計算骨組みで 10 回問題を生成したとき、少なくとも 3 種類以上の異なる問題文表現（アイテム名またはシチュエーションの組み合わせが異なるもの）が生成される
- **SC-005**: レベル1〜4 の全問題パターン（計 16 パターン以上）から問題が生成できる
- **SC-006**: 年齢算・入れ子割合・つるかめ算・旅人算など逆算ロジックを要するパターンで、answer が 100% 整数になる
- **SC-007**: 存在しないレベル ID やサブカテゴリを指定したとき、エラーオブジェクトが返り、例外が未処理のまま呼び出し元に伝播しない
- **SC-008**: Web Speech API が利用可能な主要ブラウザ（Chrome, Safari, Edge）で問題文の音声読み上げが正常に機能する
- **SC-009**: 読み上げ終了からカウントダウン開始までの遅延が 0.5 秒以内である
- **SC-010**: カウントダウンの精度が ±1 秒以内である（30 秒・60 秒どちらの設定でも）

## Assumptions

- 出力言語は日本語（問題文・単位ともに日本語）とする
- 問題文の読み上げには Web Speech API（ブラウザ標準）を使用し、音声読み上げ速度（`rate`）のデフォルトは **1.0**（スライダー初期値も 1.0）とし、変更可能とする
- 読み方のデフォルトは **「区切る」**（句読点・助詞ごとに間を置いて読み上げる）とする
- 読み上げ言語は `ja-JP` を指定する
- GUI（問題提示画面・カウントダウン表示・回答入力フォーム）はスコープに含む
- 採点・回答収集・スコア管理機能はスコープ外とする（問題生成と提示のみ）
- 問題は1問ずつ独立して生成される（連続した問題セットのセッション管理はスコープ外）
- アイテム名辞書と人物名辞書はプログラムに同梱するハードコードデータとする（外部 API 不使用）
- 濃度算（4-4）の答えは整数または小数点以下1桁とし、小数点以下2桁以上の循環小数は棄却する
- WAIS-IV の正式な心理検査としての使用を目的としておらず、学習・練習ツールとして位置づける
- レベル4「4-1 速度複合」の具体的なテンプレートは実装フェーズで詳細設計するが、逆算ロジックにより整数解を保証するという制約は必須とする
- Web Speech API が利用できない環境ではテストを実施しない（代替音源やテキスト表示のみによる代替提供はスコープ外）
